Ory Network supports a wide range of authentication methods through Ory Kratos Identities. Here's an overview:

# Authentication Methods

## [Password-based](/docs/kratos/concepts/credentials/username-email-password)

Email/Username + Password: Traditional login using a password combined with an email address or username. [Configure password
login]

## [Passwordless](/docs/kratos/passwordless/passwordless)

- [Passkeys](/docs/kratos/passwordless/passkeys): Secure login using public-key cryptography, supported by most modern browsers
  (e.g., biometric authentication like fingerprint or face recognition). [Enable passkeys]
- [One-Time Code (OTC/OTP)](/docs/kratos/passwordless/one-time-code): A two-step process where users enter their email or phone
  number and then a code sent to them. [One-time code auth]

## Social Sign-In / OIDC

OIDC (OpenID Connect): Sign in via social providers such as Google, GitHub, Facebook, and others. [Social sign-in]

## [Multi-Factor Authentication (MFA)](/docs/network/kratos/mfa-overview)

Ory supports several second-factor methods:

- TOTP (Time-Based One-Time Password): Works with apps like Google Authenticator.
- WebAuthn / FIDO2: Hardware tokens (e.g., YubiKey) or OS-level biometrics.
- Lookup Secrets: Backup/recovery codes used as a 2FA fallback.
- SMS: One-time password sent via text message.
- Email OTP: One-time code sent to the user's email as a second factor.

## [Enterprise SSO](/docs/polis/saml-federation)

SAML/OIDC SSO: Enterprise single sign-on via identity providers like Okta, Azure AD, and Google Workspace, powered by Ory Polis.
[Enterprise SSO]

You can enable or disable any of these methods via the Ory Console UI or the Ory CLI using the identity configuration. [Login
flows]
