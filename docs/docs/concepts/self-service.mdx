## Overview

Ory offers self service identity management for end-users. 
This includes:

- [Registration](./registration.mdx) using passwords, social sign in, OpenID Connect, ...;
- Login with passwords, social sign in, OpenID Connect, ...;
- Updating the profile, email, changing the password, un/linking with Social Sign In providers, ...;
- Recovering the account by resetting the password;
- Verifying email addresses, phone numbers, ...;


While we are working on improving the Ory documentation, please head over
to our open source documentation for a deep-dive on this topic:

https://www.ory.sh/kratos/docs/self-service

Currently, it is not possible to customize the UI in Ory but we expect
this feature to land soon.


## Glossary

An (not exhaustive) overview of terminology used in the Ory documentation.  
If you think a term is missing here, please [let us know](https://github.com/ory/docs/issues/new).

### Basics
`Browser Client`: User side. A browser client refers to the Web browser in the user's machine or mobile device.

`API Client`: User side. A browser client refers to the Web browser in the user's machine or mobile device.

### Flows

`Registration Flow`: The path that a user has to take in order to create an identity at your application.

`Login Flow`: The path that a user has to take in order to authenticate (i.e. prove they are a certain identity) at your application.

### Ory specific Terms

`Ory Project`: A single deployment of Ory. Each project runs their own copy of the Ory services.

`Ory Project Settings`: The configuration for each individual Ory Project can be changed in the Ory Project Settings.

`Ory Login Session`: A cookie or token stored in the users Browser or API Client. It represents a successful Login Flow and lets the user access the authenticated identity.

`$ORY_PROJECT_SLUG`: The generic URL that is generated when a project is created, e.g. https://suspicious-curran-y23rz7k2t.projects.oryapis.com

`$ORY_PROJECT_PUBLIC_API`: The endpoint for accessing your projects public API, it is composed like so: https://$ORY_PROJECT_SLUG/api/kratos/public

`$ORY_PROJECT_ADMIN_API`: The endpoint for accessing your projects administrative API, it is composed like so: https://$ORY_PROJECT_SLUG/api/kratos/admin
