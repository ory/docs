## v26.2.18

### Configurable retention for SCIM webhook event logs

You can now set how long SCIM (directory sync) webhook event logs are kept, using the `DSYNC_WEBHOOK_LOGS_TTL` environment
variable. Set it to a duration such as `720h` or `30d`.

If you leave the variable unset, logs keep the existing 7-day retention. Set it to an empty string to keep logs indefinitely. An
unrecognized value falls back to the 7-day default and logs a warning, so a typo never disables retention.
