## v26.1.17

### Add UAE PASS as a social sign-in provider

Adds support for [UAE PASS](https://docs.uaepass.ae/) authentication, the UAE's official digital identity platform. Supports both
web and mobile authentication flows.

For setup instructions, see the [Ory UAE PASS guide](https://ory.sh/docs/kratos/social-signin/uaepass).

### OIDC social login link/unlink is now available in native/API settings flows

Native mobile apps and other API clients can now manage OIDC (social login) connections through the settings flow. Previously,
link and unlink nodes were only populated for browser flows. The link action returns a 422 `browser_location_change_required`
error with a redirect URL, following the same pattern used by login and registration flows.
