# Microsoft Entra ID (formerly Azure AD) SAML SSO

In this section, we will show you how to set up your own custom SAML application for Microsoft Entra ID SAML.

## Create application

From your Azure Admin console, click **Enterprise applications** from the left navigation menu.

![Microsoft Entra ID SAML Step 1](../_assets/sso-providers/azure/1.png)

If your application is already created, choose it from the list and move to the section
[Configure Application](#configure-application)

If you haven't created a SAML application, click **New application** from the top to create a new application.

![Microsoft Entra ID SAML Step 2](../_assets/sso-providers/azure/2.png)

From the next screen, click **Create your own application**. Give your application a **Name** and click **Create**.

![Microsoft Entra ID SAML Step 3](../_assets/sso-providers/azure/3.png)

## Configure application

Select **Single Sign On** from the **Manage** section of your app and then **SAML**.

![Microsoft Entra ID SAML Step 4](../_assets/sso-providers/azure/4.png)

Click **Edit** on the **Basic SAML Configuration** section.

![Microsoft Entra ID SAML Step 5](../_assets/sso-providers/azure/5.png)

Enter the following values in the **Basic SAML Configuration** section on the next screen:

- **Identifier (Entity ID)**
- **Reply URL (Assertion Consumer Service URL)**

Replace the values with the ones you have received from Ory Polis.

Click **Save** to save your changes.

![Microsoft Entra ID SAML Step 6](../_assets/sso-providers/azure/6.png)

## Attribute mapping

Click **Edit** on the **Attributes & Claims** section.

![Microsoft Entra ID SAML Step 7](../_assets/sso-providers/azure/7.png)

You have to configure the following attributes under the **Attributes & Claims** section:

| Name                                                                 | Value                  |
| -------------------------------------------------------------------- | ---------------------- |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress` | user.mail              |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/givenname`    | user.givenname         |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name`         | user.userprincipalname |
| `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/surname`      | user.surname           |

See the screenshot below.

![Microsoft Entra ID SAML Step 8](../_assets/sso-providers/azure/8.png)

Go to the section **SAML Signing Certificate** section and download the **Federation Metadata XML**.

![Microsoft Entra ID SAML Step 9](../_assets/sso-providers/azure/9.png)

## Next steps

You've successfully configured your custom SAML application for Microsoft Entra ID SAML. At this stage, you can assign users to
your application and start using it.
