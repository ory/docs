[CockroachDB](https://www.cockroachlabs.com) — distributed SQL with
horizontal scalability, strong consistency, multi-region survivability,
and PostgreSQL wire-protocol compatibility. A viable backend for
self-hosted Ory deployments needing geographic distribution or
high-availability beyond a single Postgres node.

## Pattern

Cockroach speaks Postgres wire protocol — Ory's DSN format unchanged:

```
postgres://user:password@cockroach-host:26257/ory_kratos?sslmode=verify-full&options=--cluster%3D<cluster-id>
```

Migrations run with `kratos migrate sql` (and similar for Hydra/Keto)
against Cockroach without changes.

## Cockroach-aware tuning

- **FK-heavy schema**: Ory's schema has many foreign keys; expect
  some `40001 retry` errors under high concurrency. Configure clients
  to retry transactions per Cockroach's standard pattern.
- **`SERIAL` vs `UUID`**: Ory schemas use UUIDs (good for Cockroach
  distribution). Legacy `SERIAL` would hot-shard — verify no Ory
  product version uses SERIAL.
- **Serverless vs Self-hosted**: CockroachDB Serverless has
  request-per-second limits unsuitable for production identity
  workloads. Use Self-hosted (Dedicated) tier.
