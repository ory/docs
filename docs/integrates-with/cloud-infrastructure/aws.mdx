## Two patterns

### Pattern A — AWS application talking to Ory Network

- Outbound HTTPS (port 443) from your VPC to `*.oryapis.com`. Most
  security groups need an explicit egress rule.
- Store the Ory API key in **AWS Secrets Manager** (or SSM Parameter
  Store) and inject via task / pod IAM.
- Use **Route 53** to map your custom auth domain (for example
  `auth.example.com`) to `custom.oryapis.com` (CNAME). Configure
  the custom domain on Ory Network via
  `ory update project --custom-domain`.

### Pattern B — Self-hosted Ory on EKS

- **EKS** for the cluster (`eksctl create cluster --with-oidc` is
  the simplest start).
- **RDS PostgreSQL** Multi-AZ for Kratos / Hydra / Keto datastores.
- **ElastiCache for Redis** (optional) for session caching with
  `at_rest_encryption_enabled` and `transit_encryption_enabled`.
- Deploy via the official Helm charts
  (`helm repo add ory https://k8s.ory.sh/helm/charts`).
- **AWS Load Balancer Controller** (ALB Ingress) for HTTPS
  termination with an ACM certificate.
