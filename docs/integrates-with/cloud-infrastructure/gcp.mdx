## Two patterns

### Pattern A — GCP application talking to Ory Network

- Outbound HTTPS to `*.oryapis.com` from the workload's VPC. Default
  GKE / Cloud Run egress allows this; restrict via Cloud NAT +
  Firewall rules if needed.
- Store the Ory API key in **Secret Manager**; mount via Workload
  Identity (GKE) or runtime secret references (Cloud Run).
- Use **Cloud DNS** to map your custom auth domain to
  `custom.oryapis.com` (CNAME).

### Pattern B — Self-hosted Ory on GKE

- **GKE Autopilot** or Standard with Workload Identity enabled.
- **Cloud SQL for PostgreSQL** (HA configuration). Connect via the
  Cloud SQL Auth Proxy sidecar (preferred) or private IP with VPC
  peering.
- **Memorystore for Redis** (optional).
- Deploy via the official Helm charts.
- **GKE Gateway** (or Ingress with GCLB) for HTTPS termination with
  a Google-managed certificate.
