[Steam](https://steamcommunity.com/dev) by Valve is the world's largest
PC gaming platform. Sign in with Steam is essential for gaming
platforms, game launchers, community sites, and esports apps — but the
protocol gap to OAuth 2.0 / OIDC requires a bridge component.

## Recommended approach — OpenID 2.0 → OIDC proxy

Deploy a small proxy in front of Steam's OpenID 2.0 endpoint that
exposes an OIDC-compliant interface. Configure the proxy as a generic
OIDC provider in Kratos.

The proxy receives the OIDC authorization request from Kratos,
translates to Steam's OpenID 2.0 flow, validates the response, extracts
the 64-bit Steam ID from `openid.claimed_id`, calls the Steam Web API
(`GetPlayerSummaries`) for the profile, and returns a normal
`id_token` to Kratos.

## Steam-specific quirks

- **No email** — use `steam_id` as the credential identifier in the
  identity schema.
- **No client registration** — Steam doesn't issue Client ID/Secret;
  a Steam Web API key (from
  [steamcommunity.com/dev/apikey](https://steamcommunity.com/dev/apikey))
  is all you need for the profile call.
- **Profile visibility** — some fields are blank for private profiles.
- **Rate limits** — ~100,000 calls/day per API key.
