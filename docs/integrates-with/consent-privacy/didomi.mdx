[Didomi](https://www.didomi.io) — consent management platform (CMP) for
GDPR / CCPA / ePrivacy. Sync Didomi consent state into Ory identity
metadata so downstream services can read consent decisions alongside
identity data, and fulfill data-subject requests (DSARs) via Ory's
admin API.

## Pattern

```
Didomi consent / DSAR event  →  Your webhook  →  Ory Admin API
                                  (verify HMAC,    (PATCH metadata
                                   resolve identity)  on identity)
```

1. In Didomi → **Consent Notice** → **Webhooks**, configure a webhook
   to fire on consent events with HMAC-signed payloads.
2. Deploy a small service that:
   - Verifies Didomi's HMAC signature.
   - Resolves the Ory identity by Didomi's `user_id` (matched to an
     Ory identity trait or `metadata_public.didomi_user_id`).
   - PATCHes `metadata_public.consent.didomi` on the identity.
3. For DSAR fulfillment:
   - GET `/admin/identities/{id}` for Article 15 access.
   - DELETE `/admin/identities/{id}` for Article 17 erasure.

## Notable

- DSAR responses are time-bounded (30 days under GDPR) — design for
  reliable processing, not best-effort.
- Consent strings (TCF v2 + Didomi-specific) are long; store in
  `metadata_public` only — they're not credentials.
