[Mailchimp Transactional](https://mailchimp.com/developer/transactional/)
(formerly Mandrill) — paid add-on to Mailchimp for transactional email.
Natural choice when Mailchimp marketing is already in place.

Distinct from the Mailchimp marketing-audience integration documented
under [actions/integrations/mailchimp](../../actions/integrations/mailchimp.mdx)
— this one delivers transactional emails (verification, recovery,
MFA); that one adds users to a Mailchimp list at registration.

## Quick reference

| Setting | Value |
| --- | --- |
| Host | `smtp.mandrillapp.com` |
| Port | `587` (STARTTLS), `465` (TLS), or `2525` |
| Username | Any string (`apikey` is conventional — Mandrill ignores it but it must be present) |
| Password | Mandrill API key |

Connection URI:

```
smtps://apikey:{$MD_API_KEY}@smtp.mandrillapp.com:465
```

## Notable

- Mandrill is a **paid add-on** to Mailchimp requiring a transactional
  email block purchase.
- Domain verification (SPF `include:spf.mandrillapp.com` + DKIM TXT)
  is mandatory for production sending.
- **Don't** use Mandrill's server-side templates for Ory identity
  emails — Ory renders templates before handing to SMTP.
