Sync Ory identities to Salesforce as **Leads or Contacts** via Ory
Actions, calling the Salesforce REST API with OAuth2 service-to-service
authentication.

Distinct from [Salesforce social sign-in](../../kratos/social-signin/42_salesforce.mdx)
— that integration uses Salesforce as a sign-in provider; this one
pushes Ory users into Salesforce as records.

## How it works

Ory Action on `registration.after` (async) calls
`POST /services/data/vXX.0/sobjects/Lead/` (or `Contact/`) with an
OAuth2 access token. Best practice is a token-broker service in front
of Ory that handles Salesforce token refresh.

## Setup outline

1. Salesforce → **Setup** → **App Manager** → **New Connected App**
   with OAuth Client Credentials Flow enabled.
2. Permission Set granting create access on Lead/Contact; assign to
   the integration user.
3. Ory Action on `registration.after` — Jsonnet body builds the
   Salesforce payload.

## Notable

- Connected Apps + Permission Sets require **Salesforce Enterprise
  edition or higher**.
- Daily API call limits vary by edition; high-volume signups need
  batch creation.
- Start with Lead; promote to Contact via Salesforce-side automation.
