[BankID](https://www.bankid.com) — the high-assurance electronic
identity used in Sweden and Norway. eIDAS Substantial/High assurance.
De-facto authentication for banking, public services, and healthcare
in those markets.

## Integration path

Use a broker that exposes BankID as standard OIDC; configure as a
generic OIDC provider in Kratos.

Common brokers: [Criipto](https://www.criipto.com) (cleanest),
[Signicat](https://www.signicat.com), [Nets](https://www.nets.eu).

## Setup outline

1. Pick a broker; create an OIDC application; enable Swedish and/or
   Norwegian BankID.
2. Configure Ory redirect URI as the broker's callback.
3. Configure as a generic OIDC provider in Kratos with `issuer_url`,
   client id, secret. Scopes: `openid`, `profile`, broker-specific
   personal-number scope (for example `ssn`).
4. Jsonnet maps personal number, name, country.

Personal numbers are sensitive PII — consider `metadata_admin` over
`metadata_public`. BankID satisfies PSD2 SCA.
