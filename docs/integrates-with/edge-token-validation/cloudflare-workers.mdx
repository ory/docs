[Cloudflare Workers](https://workers.cloudflare.com/) — V8-isolate
serverless runtime at the Cloudflare edge. Validate Ory Network session
tokens and JWTs at the edge so requests never round-trip to origin when
the token is invalid, and so origin services receive a trusted
`X-User-Id` header.

## How it works

1. Worker fires on every request to the Cloudflare-fronted hostname.
2. Reads the session cookie or `Authorization: Bearer` header.
3. **For JWTs (Hydra-issued)**: verifies signature against cached
   Ory JWKS using Web Crypto.
4. **For session cookies (Kratos)**: calls
   `https://<project>.projects.oryapis.com/sessions/whoami` with the
   user's session cookie. Caches the response by cookie hash for
   30–60s in Workers KV (or per-isolate memory).
5. On valid: injects `X-User-Id` header into the request before it
   proceeds to origin.
6. On invalid: returns 401 directly from the edge.

## Notable

- Workers KV adds 50–100ms on cache miss; per-isolate memory is
  faster but doesn't share across Cloudflare edges. Pick based on
  your read/write ratio.
- The 50ms CPU budget per request fits JWKS verification comfortably;
  `whoami` calls eat into the budget.
- Cloudflare Workers is the canonical reference implementation —
  [Akamai EdgeWorkers](./akamai-edgeworkers.mdx) and
  [Fastly Compute@Edge](./fastly-compute.mdx) follow the same shape
  with platform-specific adaptations.
