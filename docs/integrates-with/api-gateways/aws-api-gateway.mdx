[AWS API Gateway](https://aws.amazon.com/api-gateway/) — managed
REST/HTTP/WebSocket gateway. Validate Ory-issued JWTs at the gateway
using a **Lambda authorizer** (REST API) or a **native JWT authorizer**
(HTTP API).

## Pattern

**HTTP API (preferred)**: configure a native JWT authorizer pointing
at Ory's issuer URL — no Lambda required.

**REST API**: deploy a Lambda authorizer that verifies the JWT against
Ory's JWKS and returns an IAM policy. API Gateway caches the policy
by token for `authorizerResultTtlInSeconds`.

## Setup outline (Lambda authorizer)

1. Deploy a Lambda authorizer (Node.js or Python) that:
   - Pulls JWT from `Authorization: Bearer <token>` header.
   - Verifies against
     `https://<project>.projects.oryapis.com/.well-known/jwks.json`
     (cache in module scope, refresh on `kid` miss).
   - Returns IAM policy + principalId (= JWT `sub`).
2. Wire as Token authorizer (REST) or Lambda authorizer (HTTP).
3. Tune `authorizerResultTtlInSeconds` for cold-start vs. revocation
   latency.

## Notable

- Use the **HTTP API native JWT authorizer** when standard verification
  suffices — Lambda is only needed for custom claim-based logic.
- Cold-start latency matters — keep the authorizer minimal; use a
  lean library like `aws-jwt-verify`.
