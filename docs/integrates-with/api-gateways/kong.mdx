[Kong Gateway](https://konghq.com/) — cloud-native API gateway. Two
integration paths with Ory: JWT validation (Hydra-issued access tokens)
and Kratos session validation.

## Path A — OAuth2 JWT validation

Use Kong's stock **jwt** plugin, or community plugins (`jwt-signer`,
`jwt-keycloak`) for JWKS auto-fetch:

1. Enable the **jwt** plugin on the relevant Service / Route.
2. Configure with Ory's JWKS (or use a community plugin for auto-
   refresh).
3. Set `claims_to_verify` (typically `exp`) and `key_claim_name`.

Kong Enterprise's **OIDC** plugin handles this with JWKS auto-discovery.

## Path B — Kratos session validation

Custom plugin (or `request-transformer` + `pre-function`):

1. Plugin calls
   `https://<project>.projects.oryapis.com/sessions/whoami` with
   the user's session cookie.
2. On 200, inject identity headers (`X-User-Id`, `X-User-Email`)
   downstream and forward.
3. On 401, return 401 without forwarding.

Cache the `whoami` response per session cookie via Kong's `proxy-cache`
plugin to avoid hammering Ory on every request.

## Notable

- Rate-limit at the gateway keyed by the validated `sub` claim —
  Kong's `rate-limiting` plugin is the recommended pattern.
