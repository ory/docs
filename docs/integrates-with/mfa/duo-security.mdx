[Duo Security](https://duo.com) — workforce-focused MFA platform with
push, OTP, phone, SMS, and device trust. Common where Duo is already
the workforce-MFA standard.

## How it works

Kratos doesn't natively support Duo. Integration via an Ory Action on
`login.after` (with `can_interrupt: true`):

1. User completes primary auth; the Action fires.
2. Handler calls Duo's **Auth API**
   (`POST https://api-XXXXXXXX.duosecurity.com/auth/v2/auth`).
3. Duo prompts the user (push, OTP, phone).
4. Handler waits for response (long-poll for push, sync for OTP).
5. On Duo allow, return success; on deny, block.

For most use cases,
[native WebAuthn MFA](../../kratos/mfa/20_webauthn-fido-yubikey.mdx) is the
simpler path with equivalent or better security.

## Notable

- Duo Auth API requires Integration Key, Secret Key, and API Host
  (from the Duo Admin Panel).
- Stable username binding: map Ory identity id to Duo username at
  enrollment time.
- Duo's **Universal Prompt** uses an OIDC flow as an alternative.
