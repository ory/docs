[Workday](https://www.workday.com) — the most widely deployed enterprise
HCM and source of truth for employee data in many enterprises. Wiring
Workday → Ory keeps Ory's identity store in sync with HR ground
truth.

## Three integration patterns

**Pattern A — Workday Studio** (canonical, expensive). Workday
integration system maps Workday data into SCIM 2.0 calls against
Ory's directory-sync endpoint.

**Pattern B — mid-tier connector**. Tools4ever HelloID, Saviynt, or a
custom service pulls Workday REST/SOAP, pushes to Ory SCIM.

**Pattern C — provision through your IdP** (most common). Workday →
Okta/Entra via IdP's native Workday connector, then IdP → Ory via the
existing [Okta SCIM](../../polis/directory-sync/providers/okta.mdx) or
[Microsoft Entra SCIM](../../polis/directory-sync/providers/azure.mdx)
integration. Avoids touching Workday directly.

## What Ory provides

Standard SCIM 2.0 endpoint with bearer-token auth, accepting `Users`
and `Groups`.
