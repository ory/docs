---
id: session-lifespan
title: Session Lifespans
---

Sessions expire after a certain amount of time, the **lifespan** of the session.
Configure the session lifespan using:

````mdx-code-block
import CodeBlock from '@theme/CodeBlock'
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="cloud" label="Ory CLI" default>

```shell
ory patch identity-config <your-project-id> \\
  --replace '/session/lifespan="720h"' # 30 days
```

  </TabItem>
  <TabItem value="macos" label="Self-Hosted Ory Kratos Config" default>

```shell title=path/to/kratos/config.yml
ory patch kratos-config <your-project-id> \\
  --replace '/session/lifespan="720h"' \\
  --format yaml
```

  </TabItem>
</Tabs>
````

Per default the session cookie has the `max-age` parameter set to the specified
session lifespan. You may disable this behavior by setting:

````mdx-code-block
<Tabs>
  <TabItem value="cloud" label="Ory CLI" default>

```shell
ory patch identity-config <your-project-id> \\
  --replace '/session/cookie/persistent=false'
```

  </TabItem>
  <TabItem value="macos" label="Self-Hosted Ory Kratos Config" default>

```shell title=path/to/kratos/config.yml
ory patch kratos-config <your-project-id> \\
  --replace '/session/cookie/persistent=false' \\
  --format yaml
```

  </TabItem>
</Tabs>
````

:::note

The cookie `max-age` parameter behaves as follows:

- The browser schedules the cookie for deletion when the session ends, if
  `max-age` isn't set as part of the `Set-Cookie` header. A session ends if the
  browser terminates due to a reboot or when shutting down the browser.
- The browser keeps the cookie until `max-age` otherwise.

:::

Once the lifespan reaches its end, the user needs to sign in again.
