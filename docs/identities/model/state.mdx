# Identity state

The identity state determines whether an identity is active or not with the following states:

- `active`: the identity is allowed to perform self-service flows such as sign in.
- `inactive`: the identity can't perform self-service flows.

When an inactive identity tries to sign in, the server will respond with a UI error.
